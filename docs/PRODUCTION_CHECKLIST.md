## 🔒 セキュリティ設定（推奨）

### 1. Supabaseのセキュリティ設定

- [ ] **RLSポリシーの最終確認**
  - すべてのテーブルでRLSが有効になっているか確認
  - 管理者権限が必要な操作が適切に保護されているか確認
  - `sql/10-checks/check-rls-policies.sql`を実行して確認

- [ ] **APIキーの管理**
  - `service_role`キーが絶対に公開されていないか確認
  - 必要に応じてAPIキーをローテーション

- [ ] **認証設定の確認**
  - Supabaseダッシュボードで「Authentication」→「Settings」を確認
  - メール認証、OAuth設定が適切か確認
  - **重要**: 開発環境から本番環境への認証設定の移行が完了しているか確認
  - 詳細は [AUTH_MIGRATION_GUIDE.md](./AUTH_MIGRATION_GUIDE.md) を参照

### 2. Vercelのセキュリティ設定

- [ ] **環境変数の保護**
  - 環境変数が適切な権限で管理されているか確認
  - チームメンバーへのアクセス権限を適切に設定

- [ ] **デプロイ設定**
  - 自動デプロイが適切に設定されているか確認
  - 本番環境への直接デプロイが制限されているか確認

## 💾 バックアップ設定（重要）

### Supabaseのバックアップ

- [ ] **自動バックアップの有効化**

  2. 自動バックアップを有効化
  3. バックアップの保持期間を設定（推奨: 7日間以上）

- [ ] **手動バックアップの定期実行**

## 📊 監視・ログ設定（推奨）

### 1. Vercelの監視

- [ ] **デプロイログの確認**

- [ ] **パフォーマンス監視**
  - Vercelダッシュボードで「Analytics」を確認
  - レスポンスタイムやエラー率を監視

### 2. Supabaseの監視

- [ ] **データベースのパフォーマンス**
  - Supabaseダッシュボードで「Database」→「Performance」を確認
  - スロークエリがないか確認

- [ ] **API使用量の監視**
  - Supabaseダッシュボードで「Settings」→「Usage」を確認
  - API使用量が上限に近づいていないか確認
