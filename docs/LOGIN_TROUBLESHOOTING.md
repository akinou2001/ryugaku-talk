# ログイン問題のトラブルシューティングガイド

ログインボタンを押してもログインできない問題の解決方法です。

## 🔍 問題の症状

- ログインボタンを押してもログインできない
- エラーメッセージが表示されない
- 通常のLP（ランディングページ）に戻る

## 📋 確認手順

### ステップ1: ブラウザのコンソールでエラーを確認

1. **ブラウザの開発者ツールを開く**
   - Windows/Linux: `F12` または `Ctrl + Shift + I`
   - Mac: `Cmd + Option + I`

2. **Consoleタブを開く**

3. **ログインボタンをクリック**

4. **エラーメッセージを確認**
   - エラーメッセージが表示されている場合は、その内容をメモしてください
   - よくあるエラー：
     - `Email not confirmed` - メール確認が完了していない
     - `Invalid login credentials` - メールアドレスまたはパスワードが間違っている
     - `Network error` - ネットワークエラー
     - `infinite recursion detected in policy for relation "profiles"` (エラーコード: 42P17) - RLSポリシーの無限再帰エラー（→問題6を参照）

### ステップ2: メール確認の完了を確認

本番環境でメール確認が有効になっている場合、**メール確認が完了していないとログインできません**。

1. **登録時に送信された確認メールを確認**
   - メールボックス（スパムフォルダも確認）を確認
   - 確認メールが届いているか確認

2. **確認メール内のリンクをクリック**
   - メール内の「確認」または「Verify」リンクをクリック
   - ブラウザで認証ページが開くことを確認

3. **確認後、再度ログインを試す**

### ステップ3: Supabaseダッシュボードでユーザーを確認

1. **本番環境のSupabaseダッシュボードにログイン**
   - [Supabaseダッシュボード](https://app.supabase.com)にアクセス
   - 本番用プロジェクトを選択

2. **「Authentication」→「Users」を開く**

3. **登録したメールアドレスでユーザーが存在するか確認**
   - ユーザーが存在するか確認
   - 「Email Confirmed」が「Yes」になっているか確認

4. **ユーザーが存在しない場合**
   - 新規登録が完了していない可能性があります
   - 再度新規登録を試してください

### ステップ4: メール確認の設定を確認

本番環境でメール確認が有効になっているか確認：

1. **本番環境のSupabaseダッシュボード**で「Authentication」→「Settings」を開く

2. **「Enable email confirmations」の状態を確認**
   - **有効になっている場合**: メール確認が必要です
   - **無効になっている場合**: メール確認は不要です

3. **開発環境でテストする場合**
   - 開発環境ではメール確認を無効にしても動作します
   - 本番環境ではセキュリティのため、メール確認を有効にすることを推奨します

### ステップ5: 環境変数の確認

本番環境のVercelで環境変数が正しく設定されているか確認：

1. **Vercelダッシュボード**で「Settings」→「Environment Variables」を開く

2. **以下の環境変数が正しく設定されているか確認**：
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://[本番プロジェクトID].supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=[本番用anon key]
   ```

3. **環境変数を変更した場合**
   - 必ずデプロイを再実行してください

## 🔧 よくある問題と解決方法

### 問題1: メール確認が完了していない

**症状**: ログイン時に「メールアドレスの確認が完了していません」というエラーが表示される

**解決方法**:
1. 登録時に送信された確認メールを確認
2. メール内のリンクをクリックして認証を完了
3. 確認後、再度ログインを試す

**一時的な解決方法（開発環境のみ）**:
- 開発環境では、メール確認を無効にすることができます
- Supabaseダッシュボードで「Authentication」→「Settings」→「Enable email confirmations」を無効にする

### 問題2: メールアドレスまたはパスワードが間違っている

**症状**: 「メールアドレスまたはパスワードが正しくありません」というエラーが表示される

**解決方法**:
1. メールアドレスとパスワードを再確認
2. 大文字・小文字、スペースに注意
3. パスワードを忘れた場合は、「パスワードを忘れた場合」リンクからリセット

### 問題3: ユーザーが存在しない

**症状**: 「このメールアドレスで登録されているアカウントが見つかりません」というエラーが表示される

**解決方法**:
1. 新規登録が完了しているか確認
2. 正しいメールアドレスで登録したか確認
3. 必要に応じて、再度新規登録を試す

### 問題4: ネットワークエラー

**症状**: 「ネットワークエラーが発生しました」というエラーが表示される

**解決方法**:
1. インターネット接続を確認
2. ブラウザを再読み込み
3. しばらく時間をおいてから再度試す

### 問題5: セッションが取得できない

**症状**: ログインに成功したように見えるが、すぐにログアウトされる

**解決方法**:
1. ブラウザのCookieを確認
   - ブラウザの設定でCookieが有効になっているか確認
   - サードパーティCookieがブロックされていないか確認
2. ブラウザのキャッシュをクリア
3. シークレットモードで試す

### 問題6: RLSポリシーの無限再帰エラー

**症状**: ブラウザのコンソールに以下のエラーが表示される
```
Error fetching posts: {code: '42P17', message: 'infinite recursion detected in policy for relation "profiles"'}
```

**原因**: `profiles`テーブルのRLSポリシー内で`profiles`テーブル自体を参照しているため、無限再帰が発生しています。

**解決方法**:
1. **SupabaseダッシュボードのSQL Editorを開く**
   - 本番環境のSupabaseダッシュボードにアクセス
   - 「SQL Editor」を開く

2. **修正SQLを実行**
   - `sql/05-rls-policies/fix-profiles-rls-recursion.sql`の内容をコピー
   - SQL Editorに貼り付けて実行

3. **実行結果を確認**
   - エラーが発生していないか確認
   - 関数が正しく作成されたか確認

4. **ブラウザを再読み込み**
   - 修正後、ブラウザを再読み込みして再度ログインを試す

**注意**: この修正により、管理者チェックが`SECURITY DEFINER`関数を使用して行われるようになり、RLSポリシー内での無限再帰が回避されます。

## 🧪 デバッグ方法

### 方法1: ブラウザのコンソールでログを確認

1. ブラウザの開発者ツール（F12）を開く
2. Consoleタブを開く
3. ログインボタンをクリック
4. エラーメッセージやログを確認

### 方法2: ネットワークタブでリクエストを確認

1. ブラウザの開発者ツール（F12）を開く
2. Networkタブを開く
3. ログインボタンをクリック
4. Supabaseへのリクエストを確認
5. エラーレスポンスがないか確認

### 方法3: Supabaseダッシュボードでログを確認

1. 本番環境のSupabaseダッシュボードで「Logs」→「Auth Logs」を開く
2. ログイン試行時のログを確認
3. エラーが発生していないか確認

## 📝 チェックリスト

ログインできない問題を解決するために、以下を確認してください：

- [ ] ブラウザのコンソールでエラーメッセージを確認した
- [ ] メール確認が完了している（本番環境でメール確認が有効な場合）
- [ ] Supabaseダッシュボードでユーザーが存在することを確認した
- [ ] メールアドレスとパスワードが正しいことを確認した
- [ ] 環境変数が正しく設定されていることを確認した
- [ ] ブラウザのCookieが有効になっていることを確認した
- [ ] ネットワーク接続が正常であることを確認した
- [ ] RLSポリシーの無限再帰エラーがないか確認した（エラーコード: 42P17）
